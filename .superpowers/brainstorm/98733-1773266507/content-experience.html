Content Experience — How should audio/video content surface?

You mentioned the index will primarily publish audio and video. The current site is pure text/data. How should media integrate with the entity-driven knowledge architecture?

A

Ambient Media — Background Presence

Audio/video content is embedded within entity pages as contextual media. An entity like "Claude 4" might have an audio explainer, a benchmark video comparison, a podcast episode. Media appears as inline players within the tectonic layers — not a separate destination, but woven into the depth experience.

The /listen page becomes a persistent mini-player (like Spotify's bottom bar) that follows you across the site. Click any entity's media → it starts playing, you keep browsing.

B

Media Hub — Dedicated Feeds

Separate /listen (audio) and /watch (video) pages as full immersive experiences with their own background treatments. Deploy-feed style live tickers show latest episodes. Entity pages link to related media, but media has its own navigation flow, playlists, and discovery.

Think: YouTube meets the AaaS design system. The /listen page uses aura blobs, /watch uses tectonic plates. Each has its own depth layers.

C

Hybrid — Both Integrated and Standalone

Entity pages embed contextual media (A's approach). But media also has dedicated hub pages with curated feeds, playlists by channel, and a persistent player (B's approach). The neural graph shows media as nodes too — a podcast episode about RAG connects to the RAG skill entity, the Pinecone tool entity, etc.

Media becomes a first-class entity type alongside tools, models, agents. It participates in the relationship graph and depth layers.
