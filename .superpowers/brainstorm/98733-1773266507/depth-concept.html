Dynamic Depth UX — How should topic exploration work?

The blog currently shows flat grids of entity cards. You want layered depth with associated topics. Here are 3 approaches for how a visitor navigates from surface to deep.

A

Orbital Depth — Zoom-to-Focus

Homepage shows entities as an orbital constellation — clustered by channel, floating with aura blobs behind them. Clicking a cluster zooms into that topic, revealing related entities as connected nodes. Each click goes deeper: Channel → Entity → Specs → Related entities branch outward.

Think: star map → solar system → planet surface. Background shifts (aura → tectonic plates → bedrock grid) signal depth changes.

B

Tectonic Layers — Scroll-to-Reveal

Each page section uses a different background treatment from the playgrounds (aura blobs → kinetic bars → bedrock grid → tectonic plates). As you scroll, layers peel back. Topic associations appear as sidebar branches that expand on hover — "Exploring Claude 4? See related: Claude Code agent, RAG Retrieval skill, SWE-Bench benchmark."

Think: geological strata. Surface = trending. Dig = niche. Each layer has its own visual identity from the playgrounds.

C

Neural Graph — Click-to-Expand

Entity pages have a relationship graph (interactive canvas) showing connected entities. The current entity is center-stage on a tectonic plate; related entities orbit around it. Clicking any node transitions smoothly — the clicked entity becomes the new center, its relations expand, previous context fades to the edge. Background effects shift per entity type.

Think: Wikipedia rabbit hole, but visual. Each entity type gets its own playground background treatment. Tools = bedrock grid. Models = aura blobs. Agents = kinetic bars.

Or combine them?

These aren't mutually exclusive. Option B's layered backgrounds can coexist with C's relationship graph on entity pages. We could use A's constellation as the homepage hero and B's scroll-depth for content pages. Tell me what resonates.
